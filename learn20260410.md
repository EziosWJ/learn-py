## 一、 语法糖解密：那些看起来很拽的写法

### 1. `dict | list` 与 `**meta` (类型提示的新玩法)
*   **代码长什么样**：
    ```python
    def ok(data: dict | list, message: str = "ok", **meta: object) -> dict:
    ```
*   **解决了什么痛点**：在 Python 3.10 之前，表达“多类型”得写冗长的 `Union[dict, list]`；`**meta` 则优雅地解决了动态传递附加参数的问题，不需要预定义无数个具体参数。
*   **一句话人话理解**：`|` 就是类型提示里的“或者”符号；`**meta` 就是个“万能收纳袋”，调用时多塞进去的键值对全被它打包带走了（最后变成字典里的 `meta` 字段）。

### 2. 装饰器 `@xxx` vs Java 注解 (似是而非的表兄弟)
*   **代码长什么样**：
    ```python
    @mcp.tool(name="get_vehicle_info")
    async def get_vehicle_info(): ...
    ```
*   **解决了什么痛点**：在不修改原函数代码的情况下，给函数附加额外功能（如注册到框架中）。
*   **一句话人话理解**：
    *   **Java 注解是“被动标签”**：像便利贴，自己不干活，等着 Spring 框架启动时来扫描反射。
    *   **Python 装饰器是“主动劫匪”**：是纯粹的语法糖。定义函数的那一瞬间，Python 会把下面的函数抓过来塞进装饰器里，套娃包装后再放回去。它是立刻执行的，不需要额外的扫描器。

## 二、 核心难点突破：`yield` 与上下文管理器

### 1. `yield` 的双重人格
*   **代码长什么样**（在生命周期中）：
    ```python
    async def app_lifespan():
        await db.connect()
        try:
            yield  # 程序在这里挂起
        finally:
            await db.close()
    ```
*   **解决了什么痛点**：需要把“启动阶段”和“运行阶段”强行切开，且不能阻塞主服务。
*   **一句话人话理解**：在这里 `yield` 不是阻塞（死等），而是**“让步 / 分界线”**。它像**高速公路服务区**，进去是为了把 CPU 方向盘交出去（让步），让其他请求在主路上狂飙；直到应用关闭时，才出来做最后的清理（`finally`）。

### 2. `@asynccontextmanager` 的降维打击
*   **代码长什么样**：
    ```python
    @asynccontextmanager
    async def my_resource():
        await acquire()
        try:
            yield resource
        finally:
            await release()
    ```
*   **解决了什么痛点**：原生 Python 要支持 `with` 语法得写繁琐的 `__enter__` 和 `__exit__` 类；在异步时代（有 `await` 的地方），普通的上下文管理器直接报错。
*   **一句话人话理解**：它是一个**“自动门改装器”**，只要你在函数里画一条 `yield` 分界线，它就能把函数无脑改造成支持 `async with` 的安全门，进门执行前半段，出门执行后半段，保证资源绝对释放。

## 三、 JS 经验迁移：Python 异步的真相

> **最大的误区**：“Python 里所有 async 都得 await，那岂不是变成同步排队了？”
> **真相**：`await` 绝对不是同步阻塞，它是“挂起点”，是“让步符”。没有 `await`，Python 的事件循环就不知道什么时候该切换任务。

### 1. 事件循环：谁来启动？
*   **JS (Node.js)**：自带内置发动机，自动运行。
*   **Python**：语言本身不带，必须靠第三方引擎。**在写 Web 服务（如 FastAPI/FastMCP）时，框架底层（如 uvicorn）会自动启动事件循环，你完全不需要手动操作（不要写 `asyncio.run()`）。** 只有在写独立测试脚本时才需要手动启动。

### 2. 为什么 Python 要求必须显式 `await`？
*   JS 可以发一个 Promise 不管它（不 await），靠 `.then()` 处理。
*   Python 里如果只调用 `async def` 不 `await`，这行代码**根本不会执行**（只会返回一个协程对象躺在内存里）。Python 强制你写 `await`，是为了明确标出：“这行代码会让出 CPU，调度器你可以在这里切走去干别的事。” 切得足够快，就是异步。

### 3. `aiomysql` 到底干了啥？
*   **本质**：`pymysql` 的“异步改装版”。把所有耗时的网络 IO 操作加上了 `await` 魔法。
*   **核心三件套**：
    1.  **连接池 (`create_pool`)**：像“共享单车车库”，免去频繁 TCP 握手。
    2.  **上下文 (`async with pool.acquire()`)**：“扫码借车”，出缩进自动“还车”，防止连接泄露。
    3.  **字典游标 (`DictCursor`)**：查出来的数据自动带上字段名变成字典，告别 `row[2]` 这种反人类的取值方式。

## 四、 安全与工程化：新项目的避坑指南

### 1. 动态 `IN` 查询防 SQL 注入
*   **痛点**：`IN` 查询的参数数量是动态的，不能用 f-string 拼接（必死，SQL 注入）。
*   **解法**：动态生成占位符 + 解包传参。
    ```python
    placeholders = ", ".join(["%s"] * len(ids_list)) # 生成 "%s, %s"
    query = f"... IN ({placeholders})"
    params = [type_id, *ids_list, limit] # 用 * 把列表拆碎成一维
    db.fetch_all(query, tuple(params))   # 必须传元组
    ```

### 2. 全局单例的最佳实践
*   **疑惑**：在不同文件里 `from app.db import db`，会不会冲突或创建多个实例？
*   **真相**：Python 的 `import` 机制是“天然单例”。无论 import 多少次，内存里永远是同一个对象。连接池 (`Pool`) 虽然是单例共享的，但每次 `acquire()` 出来的 `Connection` 是物理隔离的，并发绝不串数据。

### 3. 异步项目的“黄金三条纪律”
新开一个项目融入异步组件，只需记住：
1.  **传染性原则**：底层用了 `await`，上层调用必须一路 `async def` 到顶。
2.  **认准 `aio` 前缀**：连 MySQL 用 `aiomysql`，发请求用 `httpx`。**绝对不要在异步代码里用 `requests`、`pymysql`，一用就死锁。**
3.  **拒绝同步阻塞**：在 `async def` 里，绝对不能写 `time.sleep(5)` 或没有 `await` 的耗时计算，这会卡死整个事件循环（相当于带大件行李堵死高铁门）。要用 `await asyncio.sleep()` 替代。
